## Как таблица сравнения применяется к примеру (блог)

### Анализ требований системы блога

| Требование | Приоритет | Влияние на выбор |
|------------|-----------|------------------|
| Быстрое чтение постов | Высокий | → Embedding |
| Частое обновление комментариев | Средний | → References |
| Поиск по тегам | Высокий | → Hybrid |
| Масштабируемость | Средний | → References |

### Применение критериев из таблицы 1.n

*Содержимое будет добавлено при написании курсовой*

---

## Выбор оптимального метода для примера

### Рекомендуемый метод: **Гибридный подход**

### Обоснование выбора

1. **Posts ↔ Users**: Embedding (редкие изменения пользователя)
2. **Posts ↔ Comments**: References (частые добавления)
3. **Posts ↔ Tags**: References (M:N отношение)

*Детальное обоснование будет добавлено при написании курсовой*
